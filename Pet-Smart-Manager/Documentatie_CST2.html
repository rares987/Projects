Pet Smart Manager has been developed for the purpose of providing an easy way for users to manage their pets. The software is intended to be used by the pet owners. The benefits of using this software include its ease of use and its ease of maintenance.

2. Overall Description

2.1 Product Description

Pet Smart Manager is a self-contained web application that allows users to manage their pets in various ways.

2.2 Product Functions

  • create and login to the web application
  • add, edit and delete pets
  • use an in-built calendar to manage diffent types of events in the life of the pet, such as birthdays, appointments, feeding and so on
  • add restrictions to each pet in particular
  • provides a section where the user can upload media files such as videos, gifs, pictures, audio files for each pet in particular
  • provides a section where the user can see the pet's medical history, being pulled from the calendar
  • the ability to add pets to one's friend list by specifying the target pet's name and the user's name, and the ability to remove pets from the friend list

2.3 User Classes and Characteristics

  • Guest : simple viewer of the website, has no functionalities other than being able to view the "Contact", "About", "Home" pages and the ability to create an account
  • User : user with an associated account that has all the functionalities mentioned in the previous section, tied their account

2.4 Operating Environment

The website can be accessed from any device with a connection to the Internet and a web browser installed.

2.5 Design and Implementation Constraints

Pet Smart Manager is being developed using HTML, CSS, PHP, SQLite and JavaScript.

4. System Features

4.1 Home

  • home: displays the home page of the website
  • about: displays the about page of the website
  • contact us: displays the contact page of the website
  • register: opens up a pop-up window where the user can register for an account - it's only available if the user is not logged in
  • login: opens up a pop-up window where the user can login to an account - it's only available if the user is not logged in
  • dashboard: displays the dashboard page of the website - it's only available if the user is logged in

4.2 Dashboard

  • home: displays the home page of the website
  • about: displays the about page of the website
  • contact us: displays the contact page of the website
  • log out: logs the user out of the account and returns them to the home page
  • add a pet: opens up a pop-up window where the user can add a pet to the account
  • add a pet -> name: allows the user to enter the name of the pet
  • add a pet -> browse: allows the user to browse for a picture of the pet
  • click on the picture: opens the profile of the pet

4.3 Register page

Guests can create an account and then use it to login into the application and access all the features the account existence provides.

4.4 About page

Users can view informations about the founders of the application and read a short description of our website.

4.5 Contact page

Users can send a feedback to the administrators of the website. The feedback consists in sending a message to the administrators.

4.6 Dashboard page

Users can add a pet to their account and going to their profile.

4.7 Pet Profile page

Users can delete their pet's profile, add friends of their pet, see all his friends, add/see pet's restrictions, add/see pet's medical history, add an event in the calendar(Life Event, Medical Event, Feeding Event, Other Event),
delete an event from the calendar, upload photos, audios, videos of their pets, sharing on Twitter the events of the pet, xml button that shows the life events of that pet.

5. Other Nonfunctional Requirements

5.1 Safety Requirements

The team will be working on Pet Smart Manager regularly to ensure that none of the user's data are at risk of loss.

5.2 Security Requirements

The accounts created on our page are stored in a database, the password are hashed. Our site is preventing any attack of SQL injection and cross-site scripting(XSS).
